# NLP Trader 日本語ガイド

[English README](README.md)

NLP Trader は、株価や売買量のデータと、利用してよいニュースや文章を使い、売買の
考え方を過去のデータで試すためのプログラムです。

その時点でまだ届いていなかった情報は使いません。売買の費用も含めて試し計算し、
使った設定と結果をあとから確認できる形で残します。

> このプログラムが出す結果は、すべて条件を置いた仮の試算です。利益を約束するもの
> ではなく、投資の助言でもありません。小さな見本データで良い結果が出ても、実際の
> 市場で同じ結果になるとは限りません。

## このプログラムでできること

- 株価、売買量、ニュースなどを読み込む
- 情報が実際に使えるようになった時刻を守る
- 値動きだけを使う方法、文章だけを使う方法、両方を使う方法を比べる
- 過去データを使った試し計算（バックテスト）を行う
- 手数料、買う値段と売る値段の差、売買のしにくさなどを含めて計算する
- 売買、保有、費用、資産がどれだけ減ったか、結果の記録を保存する
- `paper` などの研究用コマンドは、仮の売買目標を作るだけで、証券会社へ注文を
  送らない
- これらとは別の [kabuS API 注文機能](docs/broker.md) は、本番用に設定し、利用者が
  注文内容を明示的に確認した場合に限り、現物株の実際の注文や取消を行える

日本株向けには、日本の取引日と取引時刻を使います。必要な銘柄の一日分のデータが
全部そろうまで待ちます。そのあと最初に市場が開く時刻の価格で、仮に売買したものと
して計算します。市場が閉じた時点ではまだ届いていなかったデータを、閉じる前から
知っていたように扱うことはありません。

## まず動かしてみる

必要なものは次の二つです。

- Python 3.12 が動くパソコン
- Python の準備を簡単にする [`uv`](https://docs.astral.sh/uv/)

見本を動かすだけなら、市場データの契約、パスワード、特別な高性能パソコンは不要
です。このファイルがある場所で、次を実行します。

```bash
uv sync --locked
uv run nlp-trader validate-config --config configs/sample.yaml
uv run nlp-trader smoke --config configs/sample.yaml
```

最後のコマンドは、データの読み込みから結果の作成までをまとめて確認します。成功
すると、`run_id` という実行番号と、結果を保存した場所が表示されます。

見本で使う会社名、価格、文章は、動作確認のために作った架空のデータです。見本の
数値はプログラムが動いたことを確かめるためのもので、期待できる利益ではありません。

## 結果を見る

最初に読むファイルは次の二つです。`<run_id>` は画面に表示された実行番号に置き換え
ます。

- `reports/sample/<run_id>/research_note.md`: 人が読みやすい結果と注意点
- `reports/sample/<run_id>/run.final.json`: 使った設定、データ、プログラムの版などの記録

売買の細かい記録は `data/processed/sample/<run_id>/backtests/` にあります。作られた
データ、結果、学習済みファイルは、通常はソースコードと一緒に保存されない設定です。
秘密の情報や契約データをソースコードの保存場所に追加しないでください。

## 日本株データで試す

日本株用の設定は [`configs/japan_baseline.yaml`](configs/japan_baseline.yaml) です。
データそのものは、このプロジェクトに入っていません。自分が利用を許されているデータ
を、次の場所に用意してください。

```text
data/local/japan/assets.csv
data/local/japan/market_bars.parquet
data/local/japan/text_items.jsonl
```

それぞれの内容は次のとおりです。

- `assets.csv`: 銘柄コード、名前、業種、売買単位など
- `market_bars.parquet`: 日ごとの始値、高値、安値、終値、売買量など
- `text_items.jsonl`: 利用を許されたニュースや文章。文章を使わない最初の試し方では
  空のファイルでも構いません

文章を使わない場合は、既存のファイルを消さずに空のファイルを用意できます。

```bash
mkdir -p data/local/japan
touch data/local/japan/text_items.jsonl
wc -c data/local/japan/text_items.jsonl
```

最後の表示が `0` なら空です。すでに内容がある場合は消さずに確認してください。

設定ファイルには、データを使ってよい根拠を記入する欄が二つあります。初めは仮の
内容なので、自分の契約や利用条件が分かる記録に置き換えます。準備できたら、次を
実行します。

```bash
uv run nlp-trader validate-config --config configs/japan_baseline.yaml
uv run nlp-trader ingest-market --config configs/japan_baseline.yaml
uv run nlp-trader backtest --config configs/japan_baseline.yaml --limit 320
```

必要なファイルがない間、最初の確認コマンドが失敗するのは正常です。ファイルの列や
時刻には厳しい決まりがあります。詳しくは
[`docs/japan_baseline.md`](docs/japan_baseline.md) を参照してください。このプログラムは
販売元のデータを勝手に取得したり、ウェブサイトから集めたりしません。

日本株の試し計算では、株式分割などに合わせて後から調整した価格ではなく、入力
ファイルにあるその日の始値や終値を、仮の売買価格として使います。株式分割などに
よる値動きの調整は、その時点で分かっていた内容だけを使います。

## 追加のローカル言語モデル

言語モデルを使う文章分析は初期設定では無効です。有効にする場合は
`llama-cpp-python==0.3.34` を使い、標準モデルを
`unsloth/Qwen3.6-27B-MTP-GGUF:UD-Q4_K_XL` として扱います。必要なファイルは
`Qwen3.6-27B-UD-Q4_K_XL.gguf` です。GGUF は llama.cpp が読むローカルモデルの
ファイル形式です。

ファイルは約 17.9 GB あり、実行時にはさらにメモリを使います。32 GB 以上の
ユニファイドメモリを実用上の目安にしてください。プログラム本体や通常のテストは
モデルを自動取得しません。利用条件を確認したうえで自分でダウンロードし、設定には
ファイルそのもののパスを指定します。

インストール、固定した版、ファイルのハッシュ確認、設定、実モデルを使う確認方法は
[英語のワークフロー](docs/workflows.md#optional-local-generative-semanticevidence-annotation)
にまとめています。通常のテストだけでは実モデルが読み込めることは確認できません。
専用の実モデル確認テストを別に実行してください。

## 研究用コマンドと実際の注文は別です

見本、バックテスト、レポート作成、`paper` コマンドは、証券会社へ注文を送りません。

kabuS API を使う注文機能だけは別です。確認用の動かし方では固定された見本の返事を
使うため、実際の注文はできません。本番用の動かし方は、現物株の本当の注文や取消を
行えます。

このプロジェクトでは、Codex を開発の手助けに使ったコードも、所有者自身の
プログラムとして扱います。注文機能は所有者本人だけが使い、操作する前提です。他の
人に認証情報や操作手段を渡さないでください。これは法律上の判断ではありません。
現在の利用条件の確認と安全な管理は、利用者自身の責任です。

本番の操作は、kabuStation が動いている同じ Windows パソコンで行います。使う前に
必ず [`docs/broker.md`](docs/broker.md) を読み、接続先、注文上限、停止方法、確認手順を
確認してください。

```bash
uv run nlp-trader broker --help
```

研究の結果から自動で注文する道はありません。注文機能は、利用者が別にコマンドを
実行し、内容を明示して確認したときだけ動きます。

## よく使う設定

| 設定ファイル | 用途 | データ |
|---|---|---|
| `configs/sample.yaml` | すばやい動作確認 | 同梱された小さな架空データ |
| `configs/backtest.yaml` | 条件を厳しくした動作確認 | 同梱された小さな架空データ |
| `configs/local.yaml` | 自分の手元のデータで試すためのひな形 | 自分で用意した利用可能なデータ |
| `configs/japan_baseline.yaml` | 日本株で試すためのひな形 | 自分で用意した利用可能なデータ |

`local.yaml` と `japan_baseline.yaml` は、必要なデータを置くまでは確認に失敗します。
プログラムが販売元のデータを自動でダウンロードすることはありません。

## 大切な制限

- 過去の試し計算が良くても、将来の利益は分かりません。
- 手数料や売買のしにくさは、設定した計算上の値です。実際の市場や証券会社を完全には
  再現しません。
- どの銘柄をいつ選べたか、あとから修正された情報をどう扱うかは、用意するデータにも
  左右されます。
- 大きなデータを一度に使うと、パソコンのメモリが足りなくなることがあります。最初は
  日付、銘柄、`--limit` で範囲を小さくしてください。
- 文章を詳しく読む追加機能は初期設定では使いません。試験中に必要なファイルを
  インターネットから自動取得することもありません。
